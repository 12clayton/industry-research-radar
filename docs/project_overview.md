# Project Overview

## 项目背景

`Global_QDII_Risk_Monitor` 是一个本地投资研究辅助项目，最初用于观察 QDII、全球 ETF、黄金、美元、美债和主题资产的价格结构。随着研究需求扩展，项目新增了独立的 `Industry Trend Search Engine`，用于把行业主题研究整理成可搜索、可对比、可导出的结构化页面。

这个项目的重点不是做自动化交易系统，而是把研究判断拆解为清晰模块：行业阶段、价格确认、热度、催化事件、新闻证据、产业链、风险信号和数据质量。

## 需求来源

用户需要一个可以长期迭代的本地研究工具：

- 能输入行业关键词并生成趋势页面。
- 能横向比较多个行业，而不是只看单一行业。
- 能区分本地框架数据、市场价格数据和新闻观察数据。
- 能把页面上下文导出给 ChatGPT 做进一步解释。
- 能通过 audit scripts 检查字段完整性、语言一致性和语义质量。

## 核心问题

行业研究常见难点：

- 行业名和主题词有中英文 alias，搜索容易错配。
- 价格走势、新闻、基本面线索和产业链信息分散。
- 热点行业容易被叙事带动，但价格确认和基本验证不一定同步。
- 新闻 feed 容易混入 ticker 相关但行业不相关的内容。
- mock framework 如果没有审计，容易出现模板污染。

本项目用模块化方式解决这些问题：搜索匹配、行业数据、市场价格确认、催化框架、新闻过滤、综合判断、导出和 audit 分离维护。

## 功能模块

- **单行业搜索**：输入行业关键词，生成行业趋势页面。
- **行业雷达总览**：比较所有行业的综合状态、价格确认、新闻热度和风险状态。
- **V3 市场价格确认**：使用 ETF / 龙头股代理样本计算价格层面确认度。
- **V4.1 催化事件框架**：全行业本地 catalyst framework。
- **V4.2 实时新闻观察 Lite**：轻量抓取 Yahoo Finance ticker news，做缓存、去重和相关性过滤。
- **综合判断卡**：用固定规则生成综合状态、透支风险、关键标签和一句话解释。
- **导出分析上下文**：生成 Markdown / JSON，便于复制给 ChatGPT 做解释型分析。
- **审计脚本**：检查趋势数据、catalyst 数据、news provider 和 semantic quality。

## 当前版本能力边界

当前版本可以：

- 作为本地研究页面运行。
- 处理多行业搜索和雷达对比。
- 使用公开市场代理数据辅助观察价格确认。
- 使用本地 catalyst framework 保持行业逻辑一致。
- 使用轻量新闻观察层提供证据线索。
- 导出结构化上下文。

当前版本不做：

- 不接财报 API。
- 不接研报解析。
- 不做 AI 自动行业结论。
- 不输出直接交易指令。
- 不替代完整投研流程。

## 后续计划

后续重点应放在稳定性和展示质量，而不是继续堆功能：

- Public Demo Mode。
- 行业覆盖质量复查。
- 多新闻源标准化。
- SQLite 本地数据库。
- 组合维度的行业监控。
- 更清晰的数据质量标记。
