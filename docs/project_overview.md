# Project Overview

## 项目定位

本项目是一个 **AI 辅助行业研究雷达系统**，用于行业趋势初筛、价格确认、新闻催化跟踪、风险识别、本地研究摘要和 AI 辅助研究复核。

它不是普通看盘网页，也不是套壳 ChatGPT。系统的核心价值是先把分散的行业材料结构化，再把可解释的研究输入交给人工和 AI 继续复核。

核心流程：

行业初筛 -> 价格确认 -> 新闻催化 -> 风险识别 -> 本地摘要 -> AI 复核输入包

系统不提供直接买卖建议，所有输出仅供研究参考。

## 需求来源

行业研究常见问题包括：

- 行业主题很多，但缺少横向对比入口。
- 价格走势、新闻、催化事件、风险信号和产业链信息分散。
- 热点行业容易被叙事带动，但价格确认和基本验证不一定同步。
- 新闻 feed 容易混入 ticker 相关但行业不相关的内容。
- 直接让 ChatGPT 总结材料，容易缺少结构化证据和可追溯上下文。

本项目用模块化方式解决这些问题：先构建本地行业框架和数据状态，再输出证据包、规则摘要和 AI 复核 Prompt。

## 功能模块

- **行业雷达总览**：横向比较多个行业的趋势评分、价格确认、新闻热度、催化状态、风险等级和人工复核建议。趋势评分只用于行业初筛，不是买入建议。
- **单行业研究页**：查看单个行业的完整上下文，包括行业阶段、价格确认、新闻催化、催化主题、风险信号和产业链信息。
- **V3 市场价格确认**：使用 ETF / 龙头股代理样本计算价格层面的趋势确认。
- **V4 新闻催化跟踪**：结合本地催化框架、缓存新闻、相关性过滤和动态催化快照，观察催化是否强化、分化、降温或待验证。
- **V5.1 本地研究摘要**：基于本地规则生成趋势健康度、价格新闻共振、催化主线、风险信号、待验证问题和人工复核建议；不是大模型输出。
- **V5.2 ChatGPT 研究输入包**：不调用 API，不自动生成结论，只生成可复制给 ChatGPT 的结构化研究 Prompt。
- **V5.3 数据状态栏**：显示摘要基于什么时间的数据，包括市场、新闻、催化快照和框架更新时间。
- **Audit scripts**：检查趋势数据、催化事件语义、新闻 provider schema 和全行业语义质量。

## 两类 Markdown 的区别

- **单行业 Markdown**：原始行业上下文 / 证据包，用于保存页面材料、人工复核或外部分析。它不是 ChatGPT Prompt。
- **V5.2 Prompt**：给 ChatGPT 的研究任务书，包含本地摘要、证据、风险、待验证问题和明确输出格式。

## 推荐 Demo 流程

1. 打开行业雷达总览。
2. 观察多个行业的趋势、新闻、风险和人工复核建议。
3. 进入单行业研究页。
4. 选择半导体 / AI 算力 / CPO / 黄金中的一个行业。
5. 查看价格确认、新闻催化、风险信号和产业链上下文。
6. 展示单行业 Markdown：原始行业上下文 / 证据包。
7. 进入研究摘要 / AI 输入包页。
8. 查看 V5.3 数据状态栏。
9. 查看 V5.1 本地研究摘要。
10. 生成 V5.2 ChatGPT Prompt。
11. 说明系统不是套壳 ChatGPT，而是先做结构化数据整理，再生成 AI 研究输入包。

## 当前边界

当前版本可以：

- 作为本地 Streamlit 研究页面运行。
- 做多行业初筛和雷达对比。
- 使用公开市场代理数据辅助观察价格确认。
- 使用本地催化框架和缓存新闻提供证据线索。
- 输出单行业证据包和 ChatGPT 研究输入包。
- 通过 audit scripts 维护数据结构和语义质量。

当前版本不做：

- 不接 OpenAI API。
- 不读取 API key。
- 不创建 `.env`。
- 不输出直接交易指令。
- 不新增数据库或研究档案功能。
- 不替代完整投研流程。

## 后续方向

后续更适合围绕展示质量和稳定性迭代：

- Public Demo Mode。
- 更清晰的数据新鲜度标记。
- 更多行业覆盖质量复查。
- 多新闻源标准化。
- 更完整的前端视觉设计。
