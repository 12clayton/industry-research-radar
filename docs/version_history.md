# Version History / 项目版本演进

本项目 `industry-research-radar` 是从原项目 `Global_QDII_Risk_Monitor` 中拆分出来的独立行业研究模块。

当前 GitHub 仓库的 Git 历史从拆分节点开始，但项目功能本身经历了从 V1 到 V5.3 的逐步演进。

当前仓库包含最新可运行版本代码，但不包含真实缓存、报告、日志、备份、私有配置和历史 data 文件。

---

## V1：行业搜索原型

V1 阶段建立了单行业搜索入口，支持通过行业名称查询基础行业框架。

主要功能包括：

* 支持中英文行业别名；
* 输出行业阶段、核心驱动因素、风险信号和产业链；
* 从“输入行业名”快速生成基础行业分析框架。

目标是先完成最小可用的行业趋势查询原型。

---

## V2：行业框架扩展

V2 阶段重点扩展行业覆盖范围和中文解释能力。

主要功能包括：

* 扩展行业池；
* 增加更多子行业、催化因素和风险信号；
* 强化中文字段和行业解释；
* 提高行业页面的可读性和覆盖面。

目标是让系统从单一行业工具变成更完整的行业趋势雷达。

---

## V3：市场价格确认

V3 阶段引入市场价格代理数据，用价格行为验证行业热度。

主要功能包括：

* 引入 MA20 / MA50 / MA200 等价格确认指标；
* 增加 1M / 3M / 6M 表现；
* 增加相对强弱指标；
* 形成市场价格确认分。

目标是避免只看概念热度，而是加入市场行为验证。

---

## V4：新闻催化与动态快照

V4 阶段开始引入新闻催化、动态快照和框架复核。

主要功能包括：

* 增加新闻缓存；
* 接入 Google News RSS / Yahoo Finance 等新闻来源；
* 增加新闻相关性判断；
* 区分 high / medium / low relevance 新闻；
* 增加催化框架；
* 增加动态催化快照；
* 增加框架复核；
* 增加多种 audit 脚本。

目标是判断行业热度是否有真实新闻和催化事件支撑。

---

## V4.13 - V4.15：催化语义修复

V4.13 到 V4.15 阶段主要修复催化框架和新闻语义问题。

主要改动包括：

* 修复框架 override 展示问题；
* 修复多源新闻合并排序；
* 修复 catalyst event 语义；
* 修复白酒、黄金、AI 算力等行业的类型和影响方向误判；
* 减少 `technology / positive / high` 这类模板化误判。

目标是提高催化事件解释的可信度和行业适配性。

---

## V5.1：本地研究摘要

V5.1 阶段新增本地规则版研究摘要。

该模块基于趋势评分、价格确认、催化热度、缓存新闻和行业框架，生成系统初判。

主要输出包括：

* 趋势健康度；
* 健康度解读；
* 价格与新闻是否共振；
* 共振解读；
* 当前催化主线；
* 风险信号；
* 待验证问题；
* 人工复核建议。

V5.1 不是严格意义上的 AI Agent，而是本地规则版研究摘要生成器。

---

## V5.2：AI 研究输入包 / ChatGPT Prompt 导出器

V5.2 阶段没有接入 API，而是生成可复制给 ChatGPT 的结构化研究 Prompt。

主要功能包括：

* 不接 OpenAI API；
* 不调用模型；
* 不读取 API key；
* 生成 ChatGPT 研究输入包；
* 包含 V5.1 摘要、高相关新闻、催化主线、风险信号、待验证问题和数据说明；
* 支持下载 Markdown。

目标是让网站负责整理材料，让 ChatGPT 负责深度分析和追问。

---

## V5.3：数据状态栏

V5.3 阶段新增当前数据状态区块，用于提示摘要基于什么数据生成。

主要显示内容包括：

* 本地摘要生成时间；
* 市场价格状态；
* 新闻缓存状态；
* 最新新闻时间；
* 最近 7 天相关新闻数；
* 高相关新闻数；
* 低相关新闻数；
* 催化快照更新时间；
* 催化状态；
* 证据强度；
* 数据新鲜度。

目标是解决“当前摘要基于什么时候的数据”的问题，提高研究结果的可解释性和可复核性。

---

## 当前项目定位

当前项目是一个本地 Streamlit 行业研究雷达工具，主要用于：

* 行业趋势初筛；
* 市场价格确认；
* 新闻催化跟踪；
* 催化框架复核；
* 本地研究摘要；
* ChatGPT 研究输入包导出；
* 数据状态提示。

本项目不提供直接交易指令，不构成投资建议，所有输出均应作为研究复核材料。
