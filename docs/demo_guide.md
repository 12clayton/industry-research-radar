# Demo Guide

本指南用于演示 **AI 辅助行业研究雷达系统**。推荐把它讲成一个可解释的研究工作流，而不是普通看盘网页，也不是套壳 ChatGPT。

核心流程：

行业初筛 -> 价格确认 -> 新闻催化 -> 风险识别 -> 本地摘要 -> AI 复核输入包

系统不提供直接买卖建议，所有输出仅供研究参考和人工复核。

当前 Demo 默认使用 Streamlit 浅色主题，适合 GitHub README 截图和面试演示。Streamlit 版本是研究原型和内部展示版，不是最终正式前端；后续可以用 Figma / Next.js 做独立展示前端。

## 1. 启动页面

在项目根目录运行：

```bash
streamlit run industry_app.py
```

打开 Streamlit 提供的本地地址。

## 2. 打开行业雷达总览

进入 `行业雷达总览`。

演示重点：

- 横向观察多个行业的趋势评分、价格确认、新闻热度、催化状态、风险等级和人工复核建议。
- 说明趋势评分是行业初筛信号，不是买入建议。
- 观察哪些行业价格确认较强，哪些行业新闻热度较高，哪些行业存在过热或分化风险。
- 展示筛选、排序、图表、watchlist 和本地缓存状态。

讲解口径：

> 这一页回答“先看哪些行业”。它不是交易结论，而是把多个行业放在同一张表里，帮助快速发现值得继续研究的方向。

推荐截图：页面顶部说明、三个概览卡片和雷达表格首屏。

## 3. 进入单行业研究页

进入 `单行业搜索`，建议选择以下行业之一：

- 半导体
- AI 算力
- CPO / 光模块
- 黄金

演示重点：

- 查看行业阶段、趋势摘要和框架评分。
- 查看 V3 市场价格确认，包括价格确认分、MA breadth、相对强弱和有效样本。
- 查看 V4 新闻催化，包括新闻缓存状态、7 天相关新闻数、催化事件和动态快照。
- 查看风险信号、产业链信息、驱动因素和相关子行业。

讲解口径：

> 这一页回答“这个行业为什么值得看，证据在哪里”。它把原始行业上下文集中到一个页面，便于后续人工复核。

推荐截图：单行业研究页顶部说明、综合判断卡、价格确认摘要和新闻催化摘要。

## 4. 区分单行业 Markdown 和 V5.2 Prompt

在单行业页展开 `导出原始行业上下文 / 证据包`。

说明：

- 单行业 Markdown 是原始行业上下文 / 证据包。
- 它用于保存页面材料、人工复核或给外部分析工具参考。
- 它不是 ChatGPT Prompt，不包含明确的 AI 研究任务书。

后面在 `研究摘要 / AI 输入包` 页生成的 V5.2 Prompt，才是给 ChatGPT 的研究任务书。

## 5. 进入研究摘要 / AI 输入包页

进入 `研究摘要 / AI 输入包`。

页面工作流：

数据状态 -> V5.1 本地研究摘要 -> V5.2 ChatGPT 研究输入包

演示重点：

- 查看 V5.3 数据状态栏，确认摘要基于什么时间的数据。
- 查看市场价格状态、新闻缓存状态、最新新闻时间、7 天相关新闻数、催化快照时间和证据强度。
- 说明 V5.1 是本地规则版研究摘要，不是大模型输出。
- 查看趋势健康度、价格与新闻共振、催化主线、风险信号、待验证问题和人工复核建议。
- 生成 V5.2 ChatGPT Prompt。

讲解口径：

> 这一页回答“系统初判是什么，以及应该让 AI 复核什么”。系统先做结构化数据整理和本地规则摘要，再生成可复制给 ChatGPT 的研究输入包。V5.2 不调用 API，也不自动生成结论。

推荐截图：V5.3 数据状态栏、V5.1 本地摘要和 V5.2 Prompt 文本框。

## 6. 推荐完整演示流程

1. 打开行业雷达总览。
2. 观察多个行业的趋势、新闻、风险和复核建议。
3. 进入单行业研究页。
4. 选择半导体 / AI 算力 / CPO / 黄金中的一个行业。
5. 查看价格确认、新闻催化、风险信号和产业链上下文。
6. 展示单行业 Markdown：原始行业上下文 / 证据包。
7. 进入研究摘要 / AI 输入包页。
8. 查看 V5.3 数据状态栏。
9. 查看 V5.1 本地研究摘要。
10. 生成 V5.2 ChatGPT Prompt。
11. 说明系统不是套壳 ChatGPT，而是先做结构化数据整理，再生成 AI 研究输入包。

## 7. 推荐截图页面

- 行业雷达总览：展示横向初筛、数据状态和风险观察。
- 单行业研究页：展示完整行业上下文和原始证据包。
- AI 研究输入包页：展示数据状态、本地摘要和 ChatGPT 研究 Prompt。

说明口径：

> 当前 Streamlit 页面是研究原型和内部展示版，重点是验证产品流程、数据结构和研究逻辑。正式对外展示可以继续用 Figma / Next.js 做独立前端。

## Demo Narrative

推荐讲法：

> 这个项目是一个 AI 辅助行业研究雷达系统。它先把行业初筛、价格确认、新闻催化、风险识别和本地规则摘要整理成可解释的研究工作流，再生成 ChatGPT 研究输入包用于人工复核。它不是自动交易工具，也不是简单把问题丢给 ChatGPT；核心价值是把分散、混乱的行业主题信息整理成可审计、可导出、可复核的研究材料。
